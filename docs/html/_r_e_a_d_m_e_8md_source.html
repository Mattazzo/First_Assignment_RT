1 # Research Track I - first assignment
2 
3 The assignment requires controlling a holonomic robot in a 2d space with a simple 2d simulator, Stage.
4 The simulator can be launched by executing the command:
5 
6 ```
7 rosrun stage_ros stageros $(rospack find assignment1)/world/exercise.world
8 ```
9 
10 ### The following behaviour should be achieved
11 - 1. The robot asks for a random target, with both coordinates in the interval (-6.0, 6.0).
12 - 2. The robot reaches the target.
13 - 3. Go to step 1.
14 
15 ### Consider the following requirements
16 - A new ROS packages should be created.
17 - Two processes (ROS nodes) should be developed.
18 - You may use cpp or python for writing your code.
19 - The first process will be in charge of:
20  - calling a service for receiving a random target;
21  - making the robot reach the target.
22 - The second process will act as a Service Server, by replying to the client with a random target, having x and y in the interval (-6.0, 6.0).
23 - A target is considered reached when the distance between the robot and the target is below 0.1.
24 
25 ### Some hints
26 - The first node should implement a ROS publisher (cmd_vel, for setting the robot speed), a ROS subscriber (odom, for knowing the actual robot position) and a ROS client (for receiving the random target)
27 - The second node should implement a ROS server (for setting the random target)
28 - The position of the robot is given in the topic **odom**, by using a **nav_msgs/Odometry** (Odometry message defined in the package nav_msgs) message. This means that the x and y position of the robot may be retrieved by reading the pose.pose.position.x and pose.pose.position.y fields of the message received by the callback associated with the subscriber.
29 - For the robot control, at first, check if the position has been reached. If yes, you should call the service to retrieve the new target position. If not, you should set **cmd_vel** (a **geometry_msgs/Twist** message) depending on the difference between the target (xt) and the robot position (x) (e.g., vel_x = k* (xt - x))
